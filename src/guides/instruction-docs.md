# Instruction Documents Guide

## Overview
This guide walks you through creating clear, concise, and comprehensive instruction documents (SOPs, tutorials, how‑to guides).

### 1. Define Scope & Audience
- Identify the target audience (e.g., developers, end‑users).
- List the learning objectives.

### 2. Structure
1. **Title** – Clear and descriptive.
2. **Prerequisites** – Tools, knowledge, or environment needed.
3. **Step‑by‑Step Instructions** – Use numbered lists; each step should be a single atomic action.
4. **Verification** – How to confirm the step succeeded.
5. **Troubleshooting** – Common pitfalls and fixes.

### 3. Writing Style
- Use active voice.
- Keep sentences short (≤ 20 words).
- Highlight commands or UI elements with back‑ticks.
- Include code snippets in fenced blocks.

### 4. Review Checklist
- ✅ All steps are testable.
- ✅ No ambiguous terms.
- ✅ Consistent terminology.
- ✅ Include screenshots where helpful (use the **Insert Image** button).

### 5. Export
- Save as Markdown (`.md`).
- Optionally export to PDF via the **Export** button.
